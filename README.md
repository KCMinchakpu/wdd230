The assignment objective is to apply my prerequisite knowledge and the concepts from the lesson to create and publish my course home page. # Requirements
► I am to Create a web page named "index.html" using my own colors, font type, photo, name, and location.
► The page must have the following general structure:
    Header
    Body
    Footer
► I am to make sure I use proper folder designations for my reference files. At the end, I am to have three folders in the root of my repository named as shown here below:
    images
    scripts
    styles

► I am to Place my Cascading Style Sheets (.css) file in the styles folder, image files in the images folder, and my JavaScript (.js) file in the scripts folder.

► Use a header, main, and footer element.
► Center all the content on the page and limit the width as shown.
► Use a grid or flex display in the header to get the layout as shown.
►  Meet each of the following standard checklist items:
    
☐ The doctype is HTML5 and is at the top of your HTML page.
     Let the browsing agent know what version of HTML to render on the page.
     
☐ The charset declared (UTF-8) is declared correctly. 

☐ The viewport is declared correctly.
     One objective in this course is to develop responsive web pages and we can start with this meta tag.

☐ A meaningful title is used on the page.
     A proper title can significantly increase the search engine optimization (SEO) of the page.

☐ A meta description is provided, it is unique and doesn't possess more than 150 characters.
     Example Meta Description:  "Assignment portal for Susan Smith in WDD 230: Web Frontend Development at Brigham Young University - Idaho"

☐ The HTML document is validated through the W3C Markup Validation Service. 
    Review and identify those relevant issues in the markup.  

☐ Semantic elements are used appropriately and where feasible throughout the document.  
  Example: Use <header> and not <div id="header"> - In addition, do not use class nor id attributes unless it is necessary for styling or script.

☐ There are no broken links on the page. 

☐ All embedded or inline CSS is avoided in the HTML document.

☐ If ID attributes are used, they are unique to a page. Do not use them if you really do not need to use them.

☐ The CSS is validated through the W3C CSS Validation Service.

☐ The portrait (self or family) image is optimized to be rendered in the browser.

☐ The <img> elements have an alternative text attribute which describes the image.

☐ JavaScript code is not mixed with your HTML code inline. Use external JavaScript file(s) and only reference the file in the head section or right before the body tag is closed.

☐ The current year is placed after the © symbol (Links to an external site.) in the footer using JavaScript and the document method querySelector to complete this task.

☐ The date and time that the file was last modified is labeled and dynamically driven using JavaScript in the footer using any DOM manipulation method (getElementById, querySelector, etc.).

☐ No JavaScript errors are found and do not use the document.write() 

☐ The language (lang) attribute of your website is specified and related to the language of the current page.

☐ Headings should be used properly and in the right order. Do NOT skip heading levels. Headings are to be used to outline the content of the page and not for styling purposes.

This page should use two headings, h1 and h2.
☐ The page should pass the basic accessibility standards using the WAVE tool.
